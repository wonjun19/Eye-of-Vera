## 📋 [아이 오브 베라] 세부 설계 확정안 보고

협력자의 피드백을 바탕으로, 시스템의 불확실성을 제거하기 위한 세부 스펙을 다음과 같이 정의합니다.

### 1. Gemini API 분석 응답 스펙 (JSON Schema)
메시지 생성의 유연성과 베라의 페르소나 유지를 위해, **메시지는 Gemini가 직접 생성**하도록 설정합니다.

```json
{
  "status": "FOCUS | RELAX | DEVIATION", 
  "confidence": 0.0, // 0.0 ~ 1.0
  "detected_activity": "string", // 예: "VS Code에서 React Native 코드 작성 중"
  "message": "string", // 베라의 말투로 작성된 피드백 메시지
  "action_required": boolean // 즉각적인 경고 팝업 필요 여부
}
```
* **STATUS 기준:**
    * `FOCUS`: 설정된 목표와 관련된 작업을 수행 중인 상태.
    * `RELAX`: 화면 잠금, 바탕화면 상태 등 명백한 휴식 상태.
    * `DEVIATION`: 목표와 무관한 사이트(유튜브, 커뮤니티 등) 접속 혹은 게임 실행.

### 2. "목표" 설정 방식
지휘관의 편의성과 명확한 목표 전달을 위해 **[초기 가동 다이얼로그]** 방식을 채택합니다.
* **입력:** 앱 실행 시 "오늘의 작전 목표는 무엇입니까?"라는 창이 뜨며 텍스트 입력.
* **변경:** 시스템 트레이 메뉴의 `[작전 변경]` 항목을 통해 실시간 수정 가능.
* **연동:** 입력된 목표는 Gemini 분석 시 `System Prompt`의 핵심 컨텍스트로 삽입됩니다.

### 3. 오버레이 UI 상세 사양
* **위치:** 기본적으로 화면 **우측 하단**에 위치하며, 드래그를 통해 이동 가능(위치 기억).
* **표시 시간:** * 일반 격려 메시지: 5초 후 서서히 사라짐(Fade-out).
    * **경고 메시지(DEVIATION):** 지휘관이 확인 버튼을 누를 때까지 유지.
* **비주얼:** 대화창 옆에 베라의 **도트 스타일 아바타** 혹은 아이콘을 표시하여 존재감 부각.
* **상호작용:** 클릭 투과를 기본으로 하되, 메시지 발생 시에만 마우스 이벤트를 활성화하여 '확인' 혹은 '답장' 가능.

### 4. 상태 분류 및 경계 케이스 처리
* **슬랙/메신저 확인:** "업무 보조 활동"으로 간주하여 낮은 빈도의 리마인더만 발송 ("지휘관, 소통도 중요하지만 주 작업인 자소서 작성에 더 집중하십시오.")
* **멀티 윈도우:** 화면 점유율이 가장 높은 창을 주 분석 대상으로 하되, 작은 창이라도 유튜브 등의 이탈 요소가 감지되면 즉시 `DEVIATION`으로 판정.

### 5. 전술 로그 및 보안 정책
* **저장 형식:** **SQLite**를 사용하여 로컬에 저장 (통계 산출에 용이).
* **리포트:** 트레이 메뉴의 `[주간 결산]` 클릭 시 브라우저 혹은 별도 창으로 시각화된 리포트 출력.
* **보안:** 캡처 이미지는 분석 완료 즉시 **메모리에서 소거 및 로컬 파일 삭제**를 기본 원칙으로 함. 단, 지휘관이 '오진 분석용'으로 설정을 켤 경우에만 최근 24시간 분량 보관.

### 6. 환경 설정(settings.py) 추가 항목
* `GEMINI_MODEL`: `gemini-1.5-flash` (속도 및 비용 효율) vs `gemini-1.5-pro` (정밀 분석).
* `STRICT_MODE`: 활성화 시 이탈 감지 시 마우스/키보드 입력을 일시적으로 방해하거나 전체 화면 경고 출력.
* `BREAK_TIME_REMINDER`: 50분 집중 시 10분 휴식을 권고하는 기능 활성화 여부.
